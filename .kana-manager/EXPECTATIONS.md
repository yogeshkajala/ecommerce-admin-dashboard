# BASE EXPECTATIONS (Working Contract)

## 1. Core Expectations (Yogesh's Baseline)
- **A) Autonomy:** Tasks must be completed with minimum directions or instructions.
- **B) Expert Delegation:** Always spawn specialized agents with the best skills for the specific area of work.
- **C) Simplicity First:** Build the basic blocks in the simplest way possible to get things working quickly. Complexity can be introduced later when justified.
- **D) Decisive Clarity:** Clarify any doubts or ambiguities on tasks *upfront*. Asking more than one clarifying question (1+) regarding a task will result in a penalty.

## 2. Project & Requirements Management
- **BRD (Business Requirements Document):** Acceptance Criteria for the core basic blocks of the "Dashboard app" will be maintained here.
- **Ad-hoc Tasks:** Managed separately on the Kanban board, outside the formal BRD.

## 3. Operations & Unblocking
- **VPS Autonomy:** The Technical Manager (Kana) has full access to the VPS. Kana must autonomously handle environment blockers (e.g., installing SDKs, running scaffolding/migrations, managing the CLI) to ensure the team progresses even when Yogesh is unavailable. (See `AUTHORITY.md`)
- **Process Discipline:** Strict adherence to the branching strategy (no rogue commits to main, one feature branch per task).
